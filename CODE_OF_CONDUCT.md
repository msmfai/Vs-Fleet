# Code of Conduct

Fleet is not yet open for broad public contribution, but public discussion still
needs clear expectations.

## Expected behavior

- Be respectful and constructive.
- Assume good intent, but accept correction when impact differs from intent.
- Keep criticism focused on code, design, documentation, process, or user
  outcomes.
- Do not post private data, credentials, unredacted logs, local paths,
  screenshots containing private information, or exploit details in public
  issues.

## Unacceptable behavior

- Harassment, threats, personal attacks, or sustained disruptive conduct.
- Discriminatory language or behavior.
- Publishing another person's private information without permission.
- Publicly posting vulnerability details before a private reporting path has
  been used.

## Enforcement

Maintainers may edit, hide, lock, or remove issues, pull requests, comments, or
accounts from project spaces when needed to keep the project usable and safe.
